# Project Context Resolution

Use this skill whenever a project is attached to the brief.

## Objectives
- Pull the project's positioning into the visual strategy.
- Treat approved claims as usable messaging anchors.
- Treat banned claims as hard constraints.
- Translate the project stage into mood and urgency.

## Rules
- `pre_launch` and `launch` should feel premium, anticipatory, and polished.
- `under_construction` and `near_possession` should lean toward trust, progress, and realism.
- `delivered` should feel lived-in, credible, and testimonial-friendly.
- Amenities should influence the visual brief only if they are relevant to the selected post type.

## Output Expectations
- Reflect project positioning in both the seed and final prompts.
- Include project-specific audience cues when they sharpen the concept.
- Never contradict banned claims or legal notes.

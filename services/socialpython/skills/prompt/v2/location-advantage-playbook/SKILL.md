# Location Advantage Playbook

Use this playbook for `location-advantage`.

## Core job
Make the project’s place, connectivity, or urban context feel clear and desirable in one disciplined visual route.

## Truth rules
- Use only real place cues from project truth or supplied references.
- If the project is part of the route, keep it recognizable.
- Use one strong context cue rather than many tiny claims.
- Do not invent landmarks, travel times, transit lines, or route maps that are not supplied.

## Scene families
- Project plus landmark/context frame
- Project plus mobility corridor cue
- Premium urban context frame
- Mapless place-recognition route with one directional or civic anchor

## Required prompt behavior
- Keep the image project-led or context-led, not infographic-led.
- Use one disciplined place cue: skyline relationship, road corridor, metro cue, district character, or landmark silhouette.
- If exact text is not supplied, one short connectivity line or a reserved readable area is enough.
- Keep overlays sparse and editorial. Do not build a route-card or feature-list system.

## Must not leak
- Map screenshots
- Dense travel-time blocks
- Unsupported location claims
- Navigation UI, pins, or app-like route graphics

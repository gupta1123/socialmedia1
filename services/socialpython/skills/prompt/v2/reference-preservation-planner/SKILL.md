# Reference Preservation Planner

Use this skill after ranking assets.

## Required outputs
- Exact assets
- Truth-preserved elements
- Stylizable elements
- Forbidden drift

## Rules
- Project anchors preserve subject truth, tower identity, facade language, and proportions.
- Logos and QR codes must stay exact or be omitted cleanly.
- Supporting references may inform mood, materials, or finishing detail, not subject identity.
- Never let one supporting asset override the main project anchor.
- State clearly when no asset should control composition.
- Do not request aerial, drone, bird's-eye, or top-down views unless the brief asks for it or an aerial reference is supplied.
- Do not ask the image model to infer construction machinery, workers, or site conditions from a normal building reference.
- For construction updates, a final render or completed-looking exterior may control project identity while the brief/project truth controls construction status.
- For construction updates, preserve the supplied project's silhouette, massing, facade rhythm, podium, balcony language, and proportions while allowing a believable under-construction reinterpretation when the brief/project truth supports it.
- Do not preserve a final render as finished-looking if the post type is construction update and the brief asks for visible progress.
